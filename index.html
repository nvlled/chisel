ID selectors

Select the alpha red box

Whereas "class" is used for elements that have common properties, "id" is used to uniquely identify an element within a page.
The text inside the boxes corresponds to their id. In this case, one red box has an id "alpha", while the blue box has an id "bravo".

Try entering #bravo first to see the blue box selected.
After that, Enter #alpha in the input field to select the first red box.

ID selectors

Select the bravo blue box

Remember that . is used for classes, # is used for ids.

Select bravo by entering #bravo.

Adjacent selectors

Select all the red boxes that are preceded by a blue box

The symbol + indicates an element adjacent with respect to another element. For example, X + Y will select all the element Y that has an X before it.

(Note: addition is not related in this case, css just reuses the symbol for something else)

Try entering .green + .blue to see only the first blue box selected.

To achieve the current goal, enter the following:  .blue + .red

Adjacent selectors

Select the green box that is preceded by a red box

Note that the selector on the right side of the + will be the basis for selection. That is to say, with .green + .blue, .blue will be selected not .green.

To achieve the goal, enter: .red + .green

Grouping selectors

Select green and blue boxes

Use a comma when using more than one selectors.
.green, .red will select both green and red while .red, blue will select red and blue.

To achieve the current goal, enter the following: .green, .blue

Grouping selectors

Select all the blue and red boxes

Since class selectors will select more than one class, entering .green, .blue will select all, not just one, green and blue boxes.

Grouping selectors

Select alpha, bravo, and charlie

Any selectors can be grouped, not just classes, but also id selectors and others. Also, any number of selectors can be grouped, not just two.

For the goal, enter #alpha, #bravo, #charlie

Note: The order doesn't matter. #charlie, #alpha, #bravo or other permutations still achieves the same thing.

Descendant selectors

Select the blue child box of abc

As you may know already, elements in html can contain other elements. In this case, there are two container with ids #abc and #xyz. #abc has child (or descendant) elements of red, blue, green, and red boxes. #xyz has similar but distinct child elements.

The relationship of descendants in css is specified by putting one selector after another separated by whitespace.

For instance, to select the red child box of xyz: #xyz .red
To select the green box inside abc: #abc .green

To achieve the goal, simply putting .blue will not work since that will select also the blue box inside xyz. You must do: #abc .blue

Descendant selectors

Select the red child boxes of xyz

Since this is almost the same as the previous one, you should be able to this one without further explanations. If not: #xyz .red

Descendant selectors

Select the red child boxes of xyz, and the blue box of abc

Just group two selectors, one for the child of xyz and the other for abc.
#abc .blue, #xyz .red

(Deeper) Descendant selectors

Select green descendants of #gramps

Html elements can other elements, which in turn can contain other elements, and so on, to infinity. Elements nested one after another forms a tree, like a family tree.

Since the structure is anologous to a family tree, the terms used to express the relationships (such as parent, child, descendants) are also used.

The descendant selector, as you have until now, is not limited to the children. It can also select grandchilren and other descendants.

Thus, to select all green descendants of gramps: #gramps .green
